Changes to ordinary query syntax

  • In CorpusSearch 1, consecutive queries had to be appended one at a time: 
    query:  ((((A function B) 
       AND (C function D)) 
       AND (E function F)) 
       AND (G function H))
    In CorpusSearch 2, queries can be appended with any logical combination of parentheses. The above query is more easily written: 
    query:     (A function B)
       AND (C function D)
       AND (E function F)
       AND (G function H)
    Old-style queries continue to be interpreted correctly.

  • Search functions that take an integer argument are now written with a space before the argument. For example, instead of: 
    query: (CODING column2 q)      ←  WRONG !
    you should now write this: 
    query: (CODING column 2 q)
    This change affects the following functions:

    • column
    • domsWords, domsWords>, domsWords<
    • iDomsNumber
    • iDomsTotal, iDomsTotal>, iDomsTotal<

Changes in coding queries

  • Coding is now performed once per boundary node, instead of once per sentence token.

  • The command "coding_query:" must now appear before the coding query.

Discontinued functions

  • iDoms_conj_switch (superseded by iDomsMod)
  • anyPrecedes (superseded by precedes)
  • print_complement (superseded by NOT)
